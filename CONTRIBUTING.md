# Contributing to Cursor & Claude Code Background Agents

Thank you for your interest in contributing! This document provides guidelines for contributing to this project.

## How to Contribute

### Reporting Issues

If you find a bug or have a suggestion:
1. Check if the issue already exists
2. Create a new issue with:
   - Clear title and description
   - Steps to reproduce (if applicable)
   - Expected vs actual behavior
   - Your environment (Cursor version, Claude version, etc.)

### Suggesting Improvements

We welcome suggestions for:
- New agent capabilities
- Improved prompts
- Better documentation
- Usage examples
- Integration improvements

### Contributing Code

1. **Fork the repository**
2. **Create a feature branch**: `git checkout -b feature/amazing-new-agent`
3. **Make your changes**
4. **Test thoroughly** - Try the agents with your changes
5. **Commit your changes**: `git commit -m 'Add amazing new agent'`
6. **Push to your branch**: `git push origin feature/amazing-new-agent`
7. **Open a Pull Request**

## Guidelines

### Agent Prompts

When adding or modifying agent prompts:
- Keep prompts clear and specific
- Include examples in the prompt
- Ensure output format is well-defined
- Make agents beginner-friendly

### Documentation

When adding new agents or features:
- Update `AGENTS.md` with full agent definition
- Add quick reference to `AGENTS-QUICK-START.md`
- Update `README.md` if needed
- Include usage examples

### Code Style

- Use clear, descriptive names
- Add comments for complex logic
- Follow existing patterns
- Keep prompts concise but complete

## Agent Development Process

1. **Define the agent's purpose** clearly
2. **Write the agent prompt** with:
   - Clear instructions
   - Output format specification
   - Usage examples
3. **Test the agent** with various scenarios
4. **Document the agent** in `AGENTS.md`
5. **Add quick reference** in `AGENTS-QUICK-START.md`
6. **Update configuration files** (`.cursorrules.txt`, `CLAUDE.md`)

## Questions?

If you have questions about contributing, feel free to:
- Open an issue for discussion
- Review existing issues and pull requests
- Check the documentation in `AGENTS.md`
